# Growth Intelligence Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build `v2.1.0 Growth Intelligence` by upgrading the user learning cockpit and the admin AI Runtime governance center.

**Architecture:** Extend the existing backend DTO/service responses instead of creating new endpoint families. The user cockpit derives growth intelligence from the current weekly aggregation, while the admin governance center combines catalog coverage with runtime metrics snapshots.

**Tech Stack:** Java 17, Spring Boot, Lombok, Maven multi-module, Vue 3, Element Plus, Vite.

### Task 4: Admin AI Governance Backend Quality Center

**Files:**
- Modify: `xiaou-ai/src/main/java/com/xiaou/ai/dto/governance/AiGovernanceOverviewResponse.java`
- Modify: `xiaou-ai/src/main/java/com/xiaou/ai/service/impl/AiGovernanceServiceImpl.java`

**Step 1: Extend response DTO**

Add nested DTOs for `RuntimeInsight`, `CoverageMatrixItem`, and `RiskItem`.

**Step 2: Inject runtime metrics collector**

Use `AiRuntimeMetricsCollector.snapshot()` to compute runtime success rate, fallback rate, parse failure rate, latency and recent invocation time.

**Step 3: Build quality score**

Combine configured Prompt coverage, Schema coverage, RAG coverage, Runtime health and risk severity into a 0-100 score and grade.

**Step 4: Build risk queue**

Generate risk items for missing Prompt config, missing Schema, missing RAG, high fallback rate, high error rate and high latency.

# v2.1.0 Growth Intelligence Design

## 背景

`v2.0.0` 和 `v2.0.1` 已经完成 AI Runtime 基座重构、AI 配置观测、治理目录和学习成长驾驶舱 2.0。`v2.1.0` 继续沿着“用户成长闭环 + AI 工程治理”的方向做一次大版本增强，版本主题定为 **Growth Intelligence**。

本次不新增独立大模块，而是在现有能力上补齐两条主线：

- 用户端：把学习驾驶舱从“进度汇总”升级为“AI 学习成长驾驶舱”，增加成长分、能力雷达、短板诊断、今日任务和 AI 复盘。
- 管理端：把 AI 治理从“目录和规则”升级为“AI Runtime 质量与治理中心”，增加质量评分、覆盖矩阵、风险队列和运行时洞察。

## 目标

1. 用户打开学习驾驶舱即可看到本周成长状态、能力结构、短板和下一步动作。
2. 后台管理员可以快速判断 AI Runtime 的 Prompt、Schema、RAG、Regression、运行观测是否健康。
3. 所有增强字段复用现有接口响应，保持旧前端字段兼容。
4. README、版本号和构建命令统一升级到 `v2.1.0`。

## 用户端设计

### 数据来源

继续复用 `LearningCockpitService` 已聚合的数据：

- OJ 周榜、总榜、AC 数
- 题库学习热力图、复习统计
- 闪卡学习统计和热力图
- 计划打卡统计
- 积分余额和签到日历
- 求职闭环目标岗位与周投入时长

### 新增响应字段

`LearningCockpitOverviewResponse` 增加：

- `growthScore`：本周成长分、等级、变化说明、达标状态和建议。
- `abilityRadar`：OJ、题库、闪卡、计划、积分五维能力雷达。
- `weaknesses`：短板诊断列表，按严重程度排序。
- `todayTasks`：今日建议任务，强调可执行动作。
- `aiReview`：AI 复盘文案，包含一句话总结、优势、风险和建议。

### 计算原则

- 成长分由总完成率、活跃天数、排名变化和模块均衡度构成。
- 能力雷达直接使用各模块完成率，避免引入额外存储。
- 短板优先识别低完成率模块、逾期复习、未签到、周榜下滑。
- 今日任务从下一步建议和短板中派生，保证页面动作和服务端策略一致。

## 管理端设计

### 数据来源

继续复用：

- `AiPromptCatalog`
- `AiStructuredOutputCatalog`
- `AiRagQueryCatalog`
- `AiRagRetrievalCatalog`
- `AiRuntimeMetricsCollector.snapshot()`

### 新增响应字段

`AiGovernanceOverviewResponse` 增加：

- `qualityScore`：AI Runtime 综合质量分。
- `qualityGrade`：A/B/C/D 健康等级。
- `runtimeInsight`：调用量、成功率、失败率、兜底率、Schema 失败率、平均耗时、最近调用时间。
- `coverageMatrix`：Prompt、Schema、RAG、Regression、Runtime 五类覆盖矩阵。
- `riskItems`：可排序风险队列，给出场景、严重程度、指标、原因和建议。

### 计算原则

- Prompt 覆盖来自 Prompt Catalog 配置完整性。
- Schema 覆盖来自结构化输出目录。
- RAG 覆盖来自 Query 和 Retrieval 目录。
- Regression 当前以结构化契约覆盖作为回归准备度近似指标，后续可替换为真实回归样例目录。
- Runtime 覆盖来自运行时快照中有调用记录的场景数量。

## 前端体验

### 用户端

`vue3-user-front/src/views/learning-cockpit/Index.vue`：

- 顶部升级为 Learning Cockpit 2.1。
- 总览卡片新增成长分和 AI 复盘。
- 新增能力雷达、短板诊断和今日任务区域。
- 保留 2.0 的周目标表、时间线、排名和自动驾驶计划。

### 管理端

`vue3-admin-front/src/views/system/ai-governance/index.vue`：

- 页面标题升级为 AI Runtime 质量与治理中心。
- 顶部指标新增质量分、运行成功率和风险数。
- 新增覆盖矩阵、运行洞察和风险队列。
- 保留原 AI 场景治理目录、质量规则和改进建议。

## 兼容性

- 新字段均提供默认值，旧数据缺失时页面降级显示空态。
- 接口路径不变：
  - `GET /user/learning-cockpit/overview`
  - `GET /admin/ai/governance/overview`
- 前端不新增第三方依赖。

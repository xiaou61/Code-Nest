# v2.2.0 文档计划

v2.2.0 的文档目标是把 Code Nest 从“能看 README”升级为“能按角色和模块持续维护的文档中心”。

## 建设原则

| 原则 | 说明 |
| --- | --- |
| 独立部署 | 文档站独立在 `docs-site/`，不影响后端和前端业务构建 |
| 保留原文 | `docs/` 继续作为资料库，文档站做结构化整理 |
| 模块化 | 每个功能模块都有固定模板，方便逐篇补齐 |
| 可验证 | 每篇开发文档都尽量给出源码位置、启动命令和验证方式 |
| 可持续 | 新功能合并前同步更新对应模块页 |

## 后续写作节奏

建议每次只深化 1 到 2 个模块：

1. 先读源码入口、路由和 PRD。
2. 补功能流程和源码位置。
3. 补数据库表和接口分组。
4. 补常见问题。
5. 构建文档站确认无断链。

# 响应体与错误码

后端统一响应体位于 `xiaou-common/src/main/java/com/xiaou/common/core/domain/Result.java`，错误码定义位于 `ResultCode.java`，异常收敛位于 `GlobalExceptionHandler.java`。

## 统一响应体

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1710000000000
}
```

| 字段 | 说明 |
| --- | --- |
| `code` | 业务状态码，成功为 `200` |
| `message` | 面向前端提示的消息 |
| `data` | 响应数据，可能为对象、数组、分页对象或 `null` |
| `timestamp` | 后端生成响应时的毫秒时间戳 |

## 分页响应

分页对象位于 `PageResult.java`。

```json
{
  "pageNum": 1,
  "pageSize": 10,
  "total": 100,
  "totalPages": 10,
  "records": [],
  "hasNext": true,
  "hasPrevious": false
}
```

## 基础错误码

| code | message | 场景 |
| --- | --- | --- |
| `200` | 操作成功 | 所有成功响应 |
| `400` | 请求参数错误 | 请求体格式错误、缺少参数、类型不匹配 |
| `401` | 未授权访问 | 标准未授权语义，实际登录态多用 `701`、`702` |
| `403` | 访问被禁止 | 标准禁止访问语义 |
| `404` | 资源不存在 | 路由或资源不存在 |
| `405` | 请求方法不支持 | HTTP method 不匹配 |
| `409` | 数据冲突 | 唯一键、状态冲突 |
| `415` | 不支持的媒体类型 | 上传或请求 Content-Type 不支持 |
| `429` | 请求过于频繁 | 频控和限流 |
| `500` | 系统内部错误 | 未捕获异常 |
| `503` | 服务不可用 | 外部依赖或服务不可用 |

## 业务错误码

| code | message | 场景 |
| --- | --- | --- |
| `600` | 业务处理失败 | `BusinessException` 默认错误码 |
| `601` | 参数校验失败 | Bean Validation、绑定失败 |
| `602` | 数据不存在 | 查询、更新、删除目标不存在 |
| `603` | 数据已存在 | 用户名、邮箱、版本号等重复 |
| `604` | 操作不被允许 | 状态不满足或无权操作 |

## 认证授权错误码

| code | message | 场景 |
| --- | --- | --- |
| `701` | Token无效 | 未提供 Token、Token 无效、无法识别 |
| `702` | Token已过期 | 过期、被顶替、被踢下线 |
| `703` | 权限不足 | Sa-Token 权限或角色不足 |
| `704` | 账户已被禁用 | 账号封禁或服务禁用 |
| `705` | 登录失败 | 用户名或密码错误 |

## 文件错误码

| code | message | 场景 |
| --- | --- | --- |
| `801` | 文件上传失败 | 上传链路异常 |
| `802` | 文件下载失败 | 下载链路异常 |
| `803` | 文件不存在 | 文件记录或对象不存在 |
| `804` | 文件类型不支持 | MIME 或后缀不在允许范围 |
| `805` | 文件大小超出限制 | 超过上传大小限制 |

## 异常处理策略

| 异常 | HTTP 状态 | 业务 code |
| --- | --- | --- |
| `BusinessException` | `200` | 异常内携带 code，默认 `600` |
| `NotLoginException` | `200` | `701` 或 `702` |
| `NotPermissionException`、`NotRoleException` | `200` | `703` |
| `DisableServiceException` | `200` | `704` |
| 参数校验和绑定异常 | `400` | `400` 或 `601` |
| 方法不支持 | `405` | `405` |
| 媒体类型不支持 | `415` | `415` |
| 文件大小超限 | `413` | `805` |
| 路由不存在 | `404` | `404` |
| 未捕获异常 | `500` | `500` |

## 前端处理建议

1. 响应拦截器先判断 `code === 200`。
2. `701`、`702` 统一清理登录态并跳转登录页。
3. `703` 显示权限不足，不应重复重试。
4. `601` 优先展示后端返回的字段校验消息。
5. 文件接口同时检查 HTTP 状态和业务 code。

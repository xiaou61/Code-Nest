# 行为准则

Code Nest 希望保持一个专业、友好、可信赖的协作环境。参与 Issue、Pull Request、讨论、评审和文档贡献时，请遵守以下准则。

完整行为准则请查看 GitHub 仓库：[CODE_OF_CONDUCT.md](https://github.com/xiaou61/Code-Nest/blob/master/CODE_OF_CONDUCT.md)

---

## 我们鼓励的行为

- 尊重不同经验水平、背景和表达方式的贡献者
- 讨论问题时聚焦事实、代码、用户价值和工程风险
- 提供可复现的信息、清晰的上下文和建设性的建议
- 在 review 中指出问题的同时说明原因和可行改进方向
- 主动同步影响范围、验证结果、风险和未完成事项

## 不接受的行为

- 人身攻击、侮辱、骚扰、歧视或恶意嘲讽
- 发布他人隐私、凭据、内部信息或未脱敏数据
- 在公开渠道披露安全漏洞细节
- 故意制造噪声、刷屏、误导性信息或破坏性改动
- 以协作之名施压他人绕过测试、审核、安全流程或许可证要求

## 冲突处理

如果讨论出现分歧，请优先：

1. 回到具体事实和用户场景
2. 明确当前目标和不变量
3. 拆分争议点，先合并无争议部分
4. 对风险较高的决策保留记录，并补充回滚方案

维护者可以对违反准则的内容进行隐藏、关闭、编辑、锁定或拒绝合并。严重或重复违规者可能被限制参与项目协作。

## 适用范围

本准则适用于：

- GitHub Issue、Pull Request、Review、Discussion
- 与本项目相关的文档、代码、发布流程和安全沟通
- 项目维护者明确代表 Code Nest 参与的公开协作场景

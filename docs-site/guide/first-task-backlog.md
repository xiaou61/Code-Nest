# 首单任务题库

这页是 [首个真实任务接入指南](/guide/first-real-task) 的配套任务库。

如果上一页回答的是“第一单应该怎么选、哪些不要先碰”，那这一页回答的是更实际的问题：

现在就要分任务了，具体可以分哪些题。

为了方便直接使用，这里把首单任务按风险分成三层：

- `L0`：强烈推荐，适合作为第一单
- `L1`：推荐，适合作为第一到第二单
- `L2`：需要带教陪跑，适合作为第二到第三单

## 使用方法

推荐按下面顺序用这页：

1. 先用 [首个真实任务接入指南](/guide/first-real-task) 判断当前新人适合接 `L0`、`L1` 还是 `L2`。
2. 再从本页挑 1 个任务。
3. 用 [带教交付模板包](/guide/onboarding-deliverable-templates) 发任务、收日报、做评审。
4. 用 [模块学习评审表](/guide/module-learning-rubric) 做结论。

## L0 任务（强烈推荐）

这一层几乎都属于“同步、校准、补齐、文档与轻量前端修正”。目标不是简单，而是闭环短、风险低、验证清晰。

### L0-01 文档站导航补入口

| 项 | 内容 |
| --- | --- |
| 任务 | 给已存在的新文档页补 sidebar 或首页入口 |
| 适合理由 | 不改业务逻辑，但能练习定位、文档同步和构建验证 |
| 预期交付 | 更新导航文件 + 一段变更说明 |
| 最低验证 | `docs-site npm run build` |
| 文档同步 | 对应路线图页、首页或模块总览 |

### L0-02 模块页互链补齐

| 项 | 内容 |
| --- | --- |
| 任务 | 给模块页补“先看什么/后看什么”的互链 |
| 适合理由 | 训练你理解页面之间的层级关系 |
| 预期交付 | 补齐至少 3 个相关文档链接 |
| 最低验证 | `docs-site npm run build` |
| 文档同步 | 模块页、模块总览 |

### L0-03 前端路由索引校准

| 项 | 内容 |
| --- | --- |
| 任务 | 校准 [前端路由索引](/reference/frontend-routes) 中某个模块的页面入口 |
| 适合理由 | 训练“从页面到文档”的最短定位闭环 |
| 预期交付 | 纠正路由、补缺失页面、写清所属模块 |
| 最低验证 | `docs-site npm run build`；如能打开页面更好 |
| 文档同步 | 前端路由索引、模块页 |

### L0-04 API 路由索引校准

| 项 | 内容 |
| --- | --- |
| 任务 | 核对某个业务域的 Controller 前缀是否与索引一致 |
| 适合理由 | 训练“从 Controller 到索引”的后端基础定位 |
| 预期交付 | 修正 API 分组索引条目 |
| 最低验证 | `docs-site npm run build`；必要时后端只读核对源码 |
| 文档同步 | API 路由索引、模块页 |

### L0-05 数据表索引补缺

| 项 | 内容 |
| --- | --- |
| 任务 | 给某个模块补一张遗漏的核心表或脚本入口 |
| 适合理由 | 训练“模块 -> 表 -> 脚本”的数据定位能力 |
| 预期交付 | 补齐表名、用途、脚本来源 |
| 最低验证 | `docs-site npm run build` |
| 文档同步 | 数据表索引、模块页 |

### L0-06 模块页验证清单补失败态

| 项 | 内容 |
| --- | --- |
| 任务 | 给某个模块的验证清单补“未登录/权限不足/空态/失败态” |
| 适合理由 | 训练验证意识，不只看 happy path |
| 预期交付 | 一组更完整的验证条目 |
| 最低验证 | `docs-site npm run build` |
| 文档同步 | 对应模块页、必要时测试与回归页 |

### L0-07 常见问题页补排障入口

| 项 | 内容 |
| --- | --- |
| 任务 | 在 [常见问题排查](/operations/troubleshooting) 中补一个故障的排障入口 |
| 适合理由 | 训练“现象 -> 入口 -> 模块”的思维 |
| 预期交付 | 一段新的排查步骤 |
| 最低验证 | `docs-site npm run build` |
| 文档同步 | 常见问题排查、问题定位流程 |

### L0-08 文档与源码不一致校准记录

| 项 | 内容 |
| --- | --- |
| 任务 | 找出并修正文档与当前源码不一致的一处描述 |
| 适合理由 | 训练“以源码为准”的习惯 |
| 预期交付 | 修正文案 + 写出校准依据 |
| 最低验证 | `docs-site npm run build` |
| 文档同步 | 对应模块页、必要时路线图或验证记录 |

## L1 任务（推荐）

这一层开始接触真实代码，但仍以“局部、可回滚、低外部依赖”为原则。

### L1-01 用户端空态文案修复

| 项 | 内容 |
| --- | --- |
| 任务 | 修复某个用户端列表页的空态或错误提示不清晰问题 |
| 适合理由 | 改动面小、容易验证、能训练前端烟测 |
| 预期交付 | 页面文案或空态组件修复 |
| 最低验证 | `vue3-user-front npm run build` + 打开真实页面 |
| 文档同步 | 如涉及路由或说明变化，同步模块页 |

### L1-02 按钮禁用或加载态补齐

| 项 | 内容 |
| --- | --- |
| 任务 | 给某个表单或提交按钮补 loading/disabled 保护 |
| 适合理由 | 能训练“重复提交”意识，但不直接碰后端幂等 |
| 预期交付 | 前端状态处理修复 |
| 最低验证 | 用户端或管理端构建 + 真实点击流程 |
| 文档同步 | 一般无需；如影响验证说明，补模块页 |

### L1-03 管理端分页或筛选重置修复

| 项 | 内容 |
| --- | --- |
| 任务 | 修复管理端列表页分页、筛选或重置逻辑 |
| 适合理由 | 改动集中，能训练管理端页面烟测 |
| 预期交付 | 列表交互修复 |
| 最低验证 | `vue3-admin-front npm run build` + 打开后台真实路由 |
| 文档同步 | 如页面行为有说明差异，补模块页或手册 |

### L1-04 表单字段校验补齐

| 项 | 内容 |
| --- | --- |
| 任务 | 给前后端已有表单补必填或格式校验 |
| 适合理由 | 能训练前后端边界，但不直接碰高风险策略 |
| 预期交付 | 前端校验/后端参数校验补齐 |
| 最低验证 | 对应前端构建 + 后端聚合构建 |
| 文档同步 | 如接口校验规则变化明显，补模块页 |

### L1-05 DTO/VO 字段说明补齐

| 项 | 内容 |
| --- | --- |
| 任务 | 补齐某个低风险模块接口 DTO/VO 的字段与文档对应关系 |
| 适合理由 | 训练接口阅读与数据结构理解 |
| 预期交付 | 字段修复或文档补充 |
| 最低验证 | 后端聚合构建 + `docs-site npm run build`（若改文档） |
| 文档同步 | API 索引、模块页、数据库字段指南（如有关） |

### L1-06 统一错误提示修正

| 项 | 内容 |
| --- | --- |
| 任务 | 修正某个页面或接口对错误码的提示文案/处理方式 |
| 适合理由 | 训练失败态和错误码理解 |
| 预期交付 | 错误处理逻辑更统一 |
| 最低验证 | 相关前端构建 + 触发失败态烟测 |
| 文档同步 | 如错误码说明变化，补 [响应体与错误码](/reference/response-errors) |

### L1-07 管理端高风险操作二次确认补齐

| 项 | 内容 |
| --- | --- |
| 任务 | 给一个删除、驳回、下架之类的按钮补二次确认 |
| 适合理由 | 能训练“风险提示”意识，但仍在低范围内 |
| 预期交付 | 二次确认交互完善 |
| 最低验证 | 管理端构建 + 真实点击验证 |
| 文档同步 | 如后台操作规范变化，补模块页 |

### L1-08 模块页与操作手册同步修复

| 项 | 内容 |
| --- | --- |
| 任务 | 修复某个模块页与用户/管理端操作手册之间的入口不一致 |
| 适合理由 | 训练“用户视角”和“开发视角”双向对照 |
| 预期交付 | 文档互相校准 |
| 最低验证 | `docs-site npm run build` |
| 文档同步 | 模块页、操作手册、首页或总览页 |

## L2 任务（需要带教陪跑）

这一层开始触碰真实后端行为，但仍然尽量回避最复杂的高风险链路。

### L2-01 后端普通 CRUD 字段补齐

| 项 | 内容 |
| --- | --- |
| 任务 | 给低风险业务模块补一两个缺失字段的读写逻辑 |
| 适合理由 | 训练后端 Controller/Service/Mapper 闭环 |
| 预期交付 | 后端字段读写修复 |
| 最低验证 | `mvn -pl xiaou-application -am clean package -DskipTests` + 对应页面或接口烟测 |
| 文档同步 | API 路由索引、模块页、必要时数据表索引 |

### L2-02 SQL 查询条件小修

| 项 | 内容 |
| --- | --- |
| 任务 | 修复一个低风险列表查询的筛选条件或排序逻辑 |
| 适合理由 | 能训练 SQL/Mapper，但影响面仍可控 |
| 预期交付 | 列表结果更符合预期 |
| 最低验证 | 后端聚合构建 + 列表页/接口烟测 |
| 文档同步 | 如筛选行为有文档说明，补模块页或手册 |

### L2-03 后端统一返回或日志补齐

| 项 | 内容 |
| --- | --- |
| 任务 | 修复某个接口返回体不统一，或补充必要日志 |
| 适合理由 | 训练工程一致性，但不改鉴权策略 |
| 预期交付 | 返回结构或日志更规范 |
| 最低验证 | 后端聚合构建 + 对应接口烟测 |
| 文档同步 | 响应体与错误码、模块页 |

### L2-04 管理端审核链路小修

| 项 | 内容 |
| --- | --- |
| 任务 | 修复一个低风险审核页的状态切换、提示或列表回流问题 |
| 适合理由 | 训练“写操作 -> 回流 -> 管理端可见”三段闭环 |
| 预期交付 | 后台审核行为更稳定 |
| 最低验证 | 管理端构建 + 后端聚合构建 + 审核流程烟测 |
| 文档同步 | 模块页、操作手册、必要时验证记录 |

## 任务分配建议（给带教同学）

推荐的分配顺序：

1. 第一个任务先从 `L0` 里选。
2. 如果 `L0` 能独立完成且交接清楚，再升到 `L1`。
3. 只有在 `L1` 输出稳定后，才给 `L2`。

别急着因为“人很聪明”就直接跳级。首单的目标不是秀操作，而是把流程跑稳。

## 每个任务至少要交什么

无论你分配的是哪一个任务，最低都建议要求下面这几项：

1. 一句话任务边界。
2. 入口/模块/接口/表至少两项定位。
3. 风险说明。
4. 最低验证结果。
5. 文档同步点。

这些内容可以直接用 [带教交付模板包](/guide/onboarding-deliverable-templates) 来收。

## 什么时候可以从题库毕业

如果学习者已经能稳定完成：

- `2` 个 `L0`
- `2` 个 `L1`
- `1` 个 `L2`

并且评审质量稳定过关，那通常就可以不再从题库里领题，而是开始接真实需求单了。

从题库毕业以后，下一步建议直接使用 [模块接手升级路线](/guide/module-ownership-ladder)，判断现在更适合继续做连续低风险交付，还是开始接一个普通模块的小闭环。

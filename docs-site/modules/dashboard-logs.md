# 仪表盘与日志

仪表盘和日志模块服务于后台运营、审计和问题定位。

## 功能入口

| 端 | 页面 |
| --- | --- |
| 管理端 | `/dashboard`、`/logs/login`、`/logs/operation` |
| 后端模块 | `xiaou-system` |

## 仪表盘

接口域：

```text
/admin/dashboard/overview
```

仪表盘通常聚合：

- 用户数据。
- 内容数据。
- 运营数据。
- 系统健康度。
- 模块关键指标。

## 登录日志

管理端登录相关接口在 `/auth` 和 `/logs` 相关控制器中。登录日志用于定位账号安全问题，包括登录时间、来源、状态、失败原因等。

## 操作日志

操作日志用于追踪管理员关键操作，例如用户状态变更、内容删除、配置调整、文件迁移、AI 配置测试等。

## 开发注意

- 高风险管理操作要记录操作日志。
- 日志清理要支持按天数清理。
- 日志查询要有分页和筛选。
- 不要在日志中记录明文密码、Token、密钥和完整敏感内容。

# 科技热点速览模块 PRD v1.0.0

## 📋 项目概述

### 🎯 项目背景

Code Nest 目前已经具备学习、求职、博客、社区、通知、AI摘要等多类能力，但还缺少一个能够持续吸引用户回访的“高频内容入口”。

当前科技资讯消费存在几个明显问题：

- **信息分散**：国内科技新闻、海外科技新闻、开源动态、AI 行业快讯分散在不同网站和平台里
- **语言门槛**：大量高价值海外资讯发布更早，但中文用户获取存在明显延迟，且阅读原文成本较高
- **结论不够快**：即使看到了新闻标题，用户仍然需要花时间判断“这条新闻到底值不值得看”
- **缺少持续订阅关系**：平台有内容承载能力，但还没有形成“每天来这里看 5 分钟科技速览”的稳定用户心智
- **AI 能力没有进入资讯消费场景**：目前 AI 更多服务于学习、面试、社区摘要，缺少面向资讯流的翻译、摘要、结构化速览能力

因此，需要新增一个独立的“科技热点速览”模块，把国内外前沿科技新闻聚合为一条中文可读、AI 可解释、后续可订阅分发的内容流。

### 💡 核心价值

- **把全球科技资讯聚合到一个入口**：减少用户在多个平台之间来回切换的成本
- **把海外内容自动中文化**：降低国外新闻阅读门槛，让用户第一时间看到可读的中文内容
- **把新闻变成可快速决策的信息**：通过 AI 速览告诉用户“这条新闻是什么、为什么重要、可能影响什么”
- **为后续订阅和日报能力铺路**：先完成页面和飞书 / 钉钉渠道接入，再逐步扩展到自动推送、精选日报、个性化推送
- **增强平台产品感与回访频次**：让 Code Nest 不只是在“有需求时才打开”的工具平台，也成为日常浏览的内容入口

### 🏠 模块定位

本功能名称为：

- **科技热点速览**

它是一个**独立页面、独立内容流、独立扩展方向**的新模块，不挂靠摸鱼工具，也不作为其他模块的附属功能。

建议产品定位如下：

- 用户端独立一级页面：`/tech-briefing`
- 管理端独立运营入口：`/admin/tech-briefing`
- 后端建议新增独立能力域或独立 Maven 模块：
  - **推荐命名**：`xiaou-tech-news`
  - 如果一期不新增模块，也至少应在后端包结构中独立为 `techbriefing` 或 `technews` 领域，避免继续挂在 `xiaou-moyu`

该模块的长期目标不是“全网热榜搬运”，而是：

- **国内外前沿科技资讯聚合器**
- **中文科技速览入口**
- **AI 摘要和翻译驱动的资讯产品**
- **后续科技日报 / 飞书钉钉订阅 / 主题订阅的产品底座**

### 🔗 模块关联

- **xiaou-ai**：承接新闻翻译、AI 速览总结、后续日报生成等能力
- **xiaou-notification**：承接后续飞书 / 钉钉消息推送、订阅任务、消息中心联动
- **xiaou-user**：承接登录用户身份、订阅主体关联与权限控制
- **xiaou-common**：承接统一枚举、缓存、线程池、定时任务、HTTP 工具等公共能力
- **vue3-user-front**：承接独立资讯流页面与详情页
- **vue3-admin-front**：承接内容源管理、抓取状态、订阅管理、文章运营
- **xiaou-moyu**：只复用已有“热榜抓取 + 定时刷新 + 缓存降级”的经验，不复用产品形态

## 🚀 一期目标与边界

### 一期目标

一期目标是先把“独立页面 + 国内外聚合 + 自动中文化 + AI 速览 + 飞书 / 钉钉订阅接入”这条链路跑通，形成可演示、可扩展、可持续运营的第一版产品。

### 一期范围

- 新增独立页面 `科技热点速览`
- 支持国内科技资讯与海外科技资讯双源聚合
- 支持对海外新闻自动生成：
  - 中文标题
  - 中文摘要
  - 中文全文机翻
- 支持生成每条新闻的 `AI 速览`
- 支持列表页和详情页两级内容浏览
- 支持分类筛选、来源筛选、时间筛选、关键词检索
- 支持飞书 / 钉钉 Webhook 订阅接入
- 支持基础管理端能力：
  - 内容源配置
  - 抓取状态查看
  - 文章内容查看
  - 订阅渠道管理

### 一期非目标

- 不做真正的“每日自动推送发送”，该能力放到二期
- 不做复杂个性化推荐算法
- 不做用户评论、点赞、转发等内容社区化功能
- 不做 App 推送、短信推送等多渠道分发
- 不做文章全文深度重写，只做翻译和速览总结
- 不做用户级新闻偏好画像建模

### 一期关键边界

- **全文机翻支持上**，但应采用“异步生成 + 缓存结果”的方式，不要求用户点击详情时同步等待翻译完成
- 对**允许抓取全文的开放来源**，详情页展示完整中文翻译
- 对**仅适合展示标题/摘要并跳转原站的来源**，允许降级为：
  - 中文标题
  - 中文摘要
  - AI 速览
  - 原文跳转
- 飞书 / 钉钉订阅一期仅完成“Webhook 绑定、测试发送、配置存储和订阅意图确认”，不做正式定时日报发送
- 一期仅支持 **机器人 Webhook 接入**，不做飞书 / 钉钉开放平台 OAuth 登录、单聊机器人授权或复杂组织权限体系

## 👤 目标用户与使用场景

### 用户角色

- **科技从业者**：关注 AI、芯片、机器人、云计算、开源和前沿产业动态
- **开发者 / 程序员**：希望快速获取高价值技术新闻，而不是刷泛娱乐热榜
- **求职冲刺用户**：希望了解行业趋势、热门技术栈、公司动向与融资消息
- **产品 / 创业 / 投资观察者**：需要快速浏览科技行业变化和热点事件
- **平台运营人员**：需要观察来源质量、抓取效果、AI 摘要效果和订阅增长

### 典型场景

#### 场景 1：早晨 5 分钟看完科技热点

用户打开 `科技热点速览` 页面，看到：

- 今日最热的 AI、芯片、机器人、开源新闻
- 每条新闻的中文标题与中文摘要
- AI 提炼的 3 条速览结论

用户不需要逐个打开多个网站，就能快速获得“今天科技圈发生了什么”。

#### 场景 2：看海外科技新闻但不想啃英文原文

用户发现一条来自海外媒体的重要资讯，例如大模型、芯片或融资新闻。

系统在详情页中展示：

- 中文标题
- AI 速览
- 中文全文机翻
- 原文链接

用户先读中文内容，再决定是否需要跳转原文深读。

#### 场景 3：判断新闻值不值得看

用户浏览列表时，不只看到标题，还能看到：

- AI 给出的“这条新闻为什么重要”
- 适合关注的人群
- 关联主题标签

从“看到标题”升级为“迅速理解价值”。

#### 场景 4：绑定飞书或钉钉，后续订阅科技日报

用户对页面认可后，在页面中绑定飞书机器人或钉钉机器人 Webhook，并选择感兴趣的方向：

- AI
- 芯片
- 开源
- 创业融资
- 消费电子

一期系统先完成渠道接入与测试推送，二期开始每天按主题推送精选科技日报。

## 🧭 产品定位与整体方案

### 核心产品形态

科技热点速览采用 **资讯流 + 详情页 + AI 中文化处理 + 订阅入口** 的整体方案：

- **列表页**：展示跨来源热点资讯流
- **详情页**：展示文章全文翻译、AI 速览和原文信息
- **AI 处理层**：负责翻译、摘要、关键词和结构化速览生成
- **订阅层**：负责飞书 / 钉钉渠道绑定、订阅偏好和后续日报分发承接
- **运营后台**：负责来源管理、抓取状态、文章运营、订阅管理

### 为什么要独立做，而不是挂在摸鱼工具里

- 摸鱼工具的产品心智偏“轻松娱乐 + 泛热榜”，而科技热点速览更偏“高价值内容产品”
- 后续还要扩展飞书 / 钉钉订阅、日报、AI 总结、主题订阅，这些都更像一个独立内容模块
- 从简历表达和产品表达上，独立模块更容易讲清楚“资讯聚合 + 翻译 + AI 摘要 + 订阅”的完整故事

### 为什么列表页与详情页都要做

- 只有列表页：用户能快速刷，但价值深度不够
- 只有详情页：产品入口弱，回访频率低
- **列表页 + 详情页**：列表负责高频消费，详情负责深度阅读和 AI 价值放大

### 为什么全文机翻和 AI 速览都要支持

- 只有标题摘要翻译：解决的是“能看懂”，但不解决“值不值得看”
- 只有 AI 速览：解决的是“看得快”，但用户仍然无法深读内容
- **全文机翻 + AI 速览同时存在**：
  - AI 速览帮助快速决策
  - 中文全文帮助真正阅读
  - 原文链接帮助权威回溯

## 📰 内容源策略

### 内容主题范围

一期建议聚焦以下前沿科技主题：

- AI / 大模型
- 芯片 / 半导体
- 机器人 / 自动驾驶
- 开源 / 开发者生态
- 云计算 / 基础设施
- 消费电子 / 智能硬件
- 创业融资 / 科技公司动态

### 内容源分类

#### 国内内容源（示例）

- 36Kr
- 极客公园
- 爱范儿
- InfoQ 中文
- 机器之心
- 钛媒体
- 掘金技术精选

#### 海外内容源（示例）

- TechCrunch
- The Verge
- VentureBeat
- Ars Technica
- Hacker News
- GitHub Trending（项目趋势）
- MIT Technology Review（可接入公开摘要）

### 接入原则

- 优先接入 **官方 RSS、开放 API、公开 Feed** 的来源
- 对于抓取限制较多、版权边界不清晰的来源，一期优先只保存：
  - 标题
  - 摘要
  - 封面
  - 来源链接
  - 关键词
- 对允许抓取正文的来源，支持全文提取与全文机翻

### 去重与聚合策略

同一科技事件可能被多个媒体同时报道，因此需要事件级去重策略：

- 根据标题相似度做粗去重
- 根据来源 URL 做强去重
- 根据发布时间和关键词做事件聚类
- 聚类后保留一个主卡片，其他报道作为“相关来源”挂在详情页

## ✨ 功能需求

### 1. 科技热点首页

#### 1.1 页面目标

让用户在 3 到 5 分钟内快速看完今日核心科技动态，并留下“值得继续看 / 值得订阅”的印象。

#### 1.2 顶部区域

- 模块标题：科技热点速览
- 副标题：一站式看懂国内外前沿科技新闻
- 刷新时间：最近一次内容更新的时间
- 运营位：可展示“今日最值得关注的主题”

#### 1.3 筛选能力

支持以下筛选维度：

- 内容范围：全部 / 国内 / 国外
- 主题分类：AI、芯片、机器人、开源、云计算、融资等
- 时间范围：1 小时 / 6 小时 / 24 小时 / 7 天
- 来源筛选：指定媒体或平台
- 排序方式：
  - 最新
  - 热度
  - 编辑推荐（预留）

#### 1.4 列表卡片信息

每条新闻卡片至少展示：

- 中文标题
- 原始来源
- 发布时间
- 主题标签
- 中文摘要
- AI 速览摘要（短版）
- 是否海外来源标识
- 是否已完成全文翻译标识
- 跳转详情按钮
- 跳转原文按钮

#### 1.5 今日焦点区

首页首屏可增加一个“今日焦点”区，展示：

- 今日最高热度新闻 1 到 3 条
- 编辑 / 系统推荐标签
- AI 给出的“今日观察点”

该区域用于提升页面第一眼产品感。

### 2. 科技热点详情页

#### 2.1 页面目标

让用户既能快速看懂新闻，也能真正深入阅读，而不需要在“AI 摘要”和“原文阅读”之间二选一。

#### 2.2 详情页核心区块

- 中文标题
- 原文标题
- 来源信息
- 发布时间
- 主题标签
- AI 速览
- 中文全文
- 原文摘要 / 原文正文片段
- 原文跳转
- 相关报道
- 订阅引导

#### 2.3 AI 速览区

AI 速览建议输出以下内容：

- 这条新闻讲了什么
- 为什么值得关注
- 可能影响哪些人 / 哪些技术方向
- 可选：一句“延伸观察”

展示形式建议为 3 到 5 条结构化 bullet。

#### 2.4 中文全文区

- 对海外新闻，支持展示中文全文机翻
- 提供“查看原文标题 / 查看原文链接”的入口
- 若全文翻译尚未完成，则展示：
  - 翻译处理中
  - 中文摘要
  - AI 速览
  - 原文跳转

#### 2.5 原文信息区

- 原始标题
- 原始来源站点
- 原始发布时间
- 原始 URL
- 如条件允许，可展示原文摘要或正文摘录

### 3. AI 能力

#### 3.1 海外内容自动翻译

对于非中文内容，一期 AI 能力至少包含：

- 标题翻译
- 摘要翻译
- 全文翻译

翻译结果建议异步生成，并存储状态：

- `PENDING`
- `PROCESSING`
- `READY`
- `FAILED`

#### 3.2 AI 速览总结

对每篇新闻生成中文 AI 速览，建议输出结构化字段：

- `summary`
- `whyImportant`
- `impactScope`
- `keywords`

#### 3.3 降级策略

当 AI 工作流不可用时：

- 海外新闻至少保留中文标题与中文摘要的规则降级
- 若全文翻译失败，则详情页仍保留：
  - 原文标题
  - 原文链接
  - AI 速览降级版或中文摘要

### 4. 飞书 / 钉钉订阅接入

#### 4.1 页面入口

首页与详情页都应提供订阅入口。

#### 4.2 订阅表单字段

- 渠道类型：飞书 / 钉钉
- Webhook 地址
- 签名密钥（可选，按渠道规则填写）
- 目标名称（群名称 / 机器人备注）
- 关注主题（多选）
- 推送频率偏好（一期仅收集，默认保留“每日”选项）
- 是否接受后续日报通知
- 是否立即发送测试消息

#### 4.3 一期能力范围

一期只完成：

- 飞书 / 钉钉 Webhook 绑定
- 测试消息发送
- 订阅偏好存储
- 重复渠道校验
- 管理端可查看 / 导出订阅列表

#### 4.4 二期承接

二期将在此基础上补齐：

- 每日自动消息推送
- 定时精选摘要
- 渠道停用 / 重新验证
- 发送日志
- 失败重试

### 5. 管理端运营能力

#### 5.1 内容源管理

- 新增 / 编辑 / 启停内容源
- 配置来源类型：国内 / 国外
- 配置抓取方式：RSS / API / 解析
- 配置抓取频率
- 配置来源权重

#### 5.2 文章管理

- 查看新闻列表
- 查看翻译状态与 AI 摘要状态
- 手动置顶 / 下线
- 手动刷新翻译 / 摘要
- 查看相关来源聚类结果

#### 5.3 任务日志

- 抓取任务执行结果
- 翻译任务执行结果
- AI 摘要任务执行结果
- 失败原因与重试次数

#### 5.4 订阅管理

## ⚠️ 风险与注意事项

### 1. 内容版权边界

- 应优先接入允许使用 RSS / API / 摘要的来源
- 对于版权边界不清晰的站点，一期避免长期缓存全文
- 详情页应保留原文链接与来源标识

### 2. 翻译成本与时效

- 全文翻译成本高于标题摘要翻译
- 必须使用异步任务和缓存结果，避免阻塞详情页访问

### 3. AI 稳定性

- AI 工作流失败时必须有规则降级
- 新闻产品对时效敏感，不应因为 AI 失败导致整条内容无法展示

### 4. 内容质量

- 需要去重、聚类、权重排序
- 否则首页会变成“多来源重复新闻堆叠”

### 5. 渠道安全

- Webhook 地址与签名密钥属于敏感信息，数据库应加密存储或至少做脱敏展示
- 管理端列表只展示脱敏后的渠道配置，不直接回显完整密钥
- 测试消息与正式推送都应记录结果，避免误发和重复发

## ✅ 总结

科技热点速览不是一个简单的热榜搬运页，而是一个面向开发者和科技从业者的独立资讯产品。它的一期核心不是做得“多”，而是先把 4 件最关键的事情做好：

- **聚合国内外高价值科技资讯**
- **让海外内容自动中文化**
- **用 AI 帮用户更快看懂新闻**
- **为后续科技日报和飞书 / 钉钉订阅打下数据与产品基础**

如果一期落地顺利，这个模块后续可以自然演进为：

- 每日科技日报
- 飞书 / 钉钉订阅产品
- AI 科技情报入口
- 平台级高频回访内容模块

# 慢SQL智能优化工作台 PRD v2.0.0

## 1. 项目概述

### 1.1 背景
现有慢SQL优化模块（v1.0）已支持单条 SQL 分析、历史记录、收藏与删除，但仍存在以下问题：

1. 用户端缺少完整工作台入口，操作路径分散。
2. 仅支持单次分析，不支持批量诊断。
3. 缺少优化前后收益对比，结果可解释性不足。
4. Coze 当前仅单工作流，无法支撑分阶段诊断与重写编排。

### 1.2 目标
构建 `SQL 智能优化工作台 2.0`，实现一站式闭环：

1. SQL 输入与格式化。
2. AI 诊断与评分。
3. 优化 SQL 生成与索引建议。
4. Before/After EXPLAIN 对比评估。
5. 历史沉淀与回放。

### 1.3 非目标

1. 不执行 SQL（仅分析文本，不连用户数据库执行）。
2. 不自动应用索引 DDL（只输出建议）。
3. 不做多人协作评论（后续版本再扩展）。

## 2. 用户价值

### 2.1 核心价值

1. 降低 SQL 调优门槛，非 DBA 也可快速定位问题。
2. 用结构化报告替代“纯文本建议”，提高可落地性。
3. 支持批量分析，贴合真实项目排查场景。
4. 提供优化收益对比，支持团队复盘和知识沉淀。

### 2.2 目标用户

1. 开发工程师（后端/全栈）。
2. 测试工程师（性能压测协同）。
3. 技术负责人（评审 SQL 质量）。

## 3. 功能需求

### 3.1 工作台首页

1. 新增用户端页面 `/sql-optimizer/workbench`。
2. 页面分区：输入区、诊断区、建议区、对比区、历史区。
3. 支持示例 SQL 一键填充和 SQL 格式化。

### 3.2 单条增强分析

1. 输入：SQL、EXPLAIN、DDL、MySQL版本。
2. 输出：
   - 评分（0-100）
   - 问题清单（类型/严重级别/定位）
   - EXPLAIN 逐行解读
   - 快速修复建议（Quick Wins）
3. 失败时返回降级结果，页面不可白屏。

### 3.3 优化重写建议

1. 自动输出优化后的 SQL。
2. 输出索引建议 DDL 列表。
3. 输出风险提示（语义变化风险、写入放大风险、兼容性风险）。
4. 支持复制 SQL/DDL。

### 3.4 优化前后对比

1. 支持用户粘贴优化后 EXPLAIN。
2. 生成收益评估：
   - 扫描行数变化
   - 访问类型变化（type）
   - Extra 变化
   - 综合收益分
3. 以卡片和表格展示差异。

### 3.5 批量分析

1. 支持一次提交 5-20 条 SQL。
2. 每条任务独立返回状态（成功/失败/降级）。
3. 支持失败项重试。
4. 支持按评分与严重级别排序。

### 3.6 历史沉淀

1. 支持分页查询历史案例。
2. 支持收藏与取消收藏。
3. 支持查看详情回放（输入、诊断结果、建议、对比结果）。

## 4. Coze 工作流要求

2.0 采用三工作流编排，详细实现写入：

- `docs/coze/SQL智能优化工作台2.0工作流配置指南.md`

工作流列表：

1. `SQL_OPTIMIZE_ANALYZE_V2`：基础诊断。
2. `SQL_OPTIMIZE_REWRITE_V2`：优化重写与索引建议。
3. `SQL_OPTIMIZE_COMPARE_V2`：优化前后收益评估。

## 5. 接口需求

### 5.1 新增接口（2.0）

1. `POST /user/sql-optimizer/workbench/analyze`
2. `POST /user/sql-optimizer/workbench/compare`
3. `POST /user/sql-optimizer/workbench/batch-analyze`
4. `GET /user/sql-optimizer/workbench/cases`
5. `GET /user/sql-optimizer/workbench/cases/{id}`

### 5.2 兼容性要求

1. 保留 v1.0 接口不变：`/user/sql-optimizer/analyze` 等。
2. 新接口独立扩展，不影响现网已有调用方。

## 6. 数据模型建议

### 6.1 快速落地方案（推荐）

1. 复用 `sql_optimize_record`。
2. 在 `analysis_result` 中增加 2.0 结构字段（rewrite/compare/batchMeta）。
3. 通过版本字段区分 `v1` 和 `v2` 记录。

### 6.2 标准化方案（后续）

1. `sql_optimize_case`：案例主表。
2. `sql_optimize_case_item`：案例子项（批量场景）。
3. `sql_optimize_case_event`：事件日志表。

## 7. 非功能需求

1. 性能：单条分析接口 P95 < 5s（不含 Coze 排队抖动）。
2. 可用性：Coze 超时/失败时必须降级返回。
3. 安全：限制输入长度、批量条数、请求体大小；禁止执行用户 SQL。
4. 可观测性：记录调用耗时、失败原因、降级次数。

## 8. 验收标准（DoD）

1. 用户端可完整走通：分析 -> 重写 -> 对比 -> 收藏 -> 历史回看。
2. 批量分析支持 10 条以上并可部分成功返回。
3. Coze 不可用时系统仍返回可解释降级结果。
4. 新接口完成基础单元测试与集成测试。
5. 文档齐全：PRD、Coze 配置指南、接口字段约定。

## 9. 里程碑

1. 回归测试与文档完善。
2. 版本发布与变更说明。

# 求职作战台（增强版）Coze工作流配置指南

本文档用于指导「求职作战台（增强版）」所需的 Coze 工作流搭建与落地。

## 概述

建议先配置 4 个核心工作流，覆盖增强版完整闭环：

| 工作流 | 建议枚举名 | 用途 |
|---|---|---|
| JD解析 | `JOB_BATTLE_JD_PARSE` | 将岗位 JD 解析成结构化要求 |
| 简历匹配评估 | `JOB_BATTLE_RESUME_MATCH` | 计算岗位匹配度并给出能力差距 |
| 补短板行动计划 | `JOB_BATTLE_PLAN_GENERATE` | 生成 30 天可执行提升计划 |
| 面试复盘总结 | `JOB_BATTLE_INTERVIEW_REVIEW` | 自动沉淀复盘结论与下一步建议 |

## 配置位置

工作流 ID 后续建议回填到：

`xiaou-common/src/main/java/com/xiaou/common/enums/CozeWorkflowEnum.java`

与当前项目一致，工作流输出请保持可被后端解析的 JSON（支持 `{"output":"..."}` 包装格式）。

---

## 工作流1：JD解析（JOB_BATTLE_JD_PARSE）

### 功能说明
把用户粘贴的 JD 文本转为结构化岗位画像，供后续简历匹配和计划生成使用。

### 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| jdText | String | 是 | 原始 JD 文本 |
| targetRole | String | 否 | 用户目标岗位（如 Java后端） |
| targetLevel | String | 否 | 目标级别（校招/初级/中级/高级） |
| city | String | 否 | 期望城市 |

### 输出格式（严格 JSON）

```json
{
  "jobTitle": "Java后端开发工程师",
  "level": "中级",
  "mustSkills": ["Java", "Spring Boot", "MySQL", "Redis"],
  "niceSkills": ["MQ", "K8s", "微服务治理"],
  "responsibilities": ["负责核心业务模块开发", "参与架构优化"],
  "seniorityYears": "3-5年",
  "keywords": ["高并发", "分布式", "性能优化"],
  "riskPoints": ["要求有大型项目经验但JD描述不清晰"],
  "summary": "该岗位偏重后端工程能力与性能优化经验"
}
```

### Coze提示词模板（可直接用）

```text
你是资深技术招聘专家，请将输入JD解析为结构化信息。

输入：
- jdText: {{jdText}}
- targetRole: {{targetRole}}
- targetLevel: {{targetLevel}}
- city: {{city}}

要求：
1. 提取岗位名称、级别、核心必备技能、加分技能、职责、关键词、风险点。
2. 若JD存在模糊描述，请放入riskPoints。
3. 输出必须是严格JSON，不要输出Markdown，不要输出额外解释。
4. 字段必须包含：
jobTitle, level, mustSkills, niceSkills, responsibilities, seniorityYears, keywords, riskPoints, summary
```

## 工作流2：简历匹配评估（JOB_BATTLE_RESUME_MATCH）

### 功能说明
根据「结构化 JD + 简历文本」输出匹配度、差距项和可执行改写建议。

### 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| parsedJdJson | String | 是 | 工作流1输出的 JSON 字符串 |
| resumeText | String | 是 | 用户简历全文 |
| projectHighlights | String | 否 | 用户补充的项目亮点 |
| targetCompanyType | String | 否 | 大厂/中厂/创业公司等 |

### 输出格式（严格 JSON）

```json
{
  "overallScore": 72,
  "dimensionScores": {
    "skillMatch": 78,
    "projectDepth": 65,
    "architectureAbility": 60,
    "communicationClarity": 80
  },
  "strengths": ["技术栈与JD重合度较高", "项目经历有量化结果"],
  "gaps": [
    {
      "skill": "分布式事务",
      "priority": "P0",
      "why": "JD明确要求，简历未体现",
      "suggestedAction": "补充一个真实项目中的事务一致性方案"
    }
  ],
  "resumeRewriteSuggestions": [
    {
      "section": "项目经历",
      "advice": "补充高并发场景下的性能指标与优化前后对比"
    }
  ],
  "missingKeywords": ["高并发", "限流", "链路追踪"],
  "estimatedPassRate": 55
}
```

### Coze提示词模板（可直接用）

```text
你是技术面试官+简历优化顾问。

输入：
- parsedJdJson: {{parsedJdJson}}
- resumeText: {{resumeText}}
- projectHighlights: {{projectHighlights}}
- targetCompanyType: {{targetCompanyType}}

任务：
1. 输出简历与JD匹配度（0-100）。
2. 分维度评分：skillMatch/projectDepth/architectureAbility/communicationClarity。
3. 给出差距项gaps，并标注优先级P0/P1/P2。
4. 给出简历改写建议和缺失关键词。
5. 输出estimatedPassRate（0-100）。

输出要求：
- 只输出严格JSON
- 不要输出额外解释文字
```

## 工作流3：补短板行动计划（JOB_BATTLE_PLAN_GENERATE）

### 功能说明
基于差距项自动生成 30 天行动清单（支持按时间预算动态调整）。

### 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| gapsJson | String | 是 | 工作流2中的 gaps 数组 |
| targetDays | Integer | 是 | 计划天数（建议 30） |
| weeklyHours | Integer | 是 | 每周可投入小时数 |
| preferredLearningMode | String | 否 | 视频/文档/实战/混合 |
| nextInterviewDate | String | 否 | 下场面试日期（yyyy-MM-dd） |

### 输出格式（严格 JSON）

```json
{
  "planName": "Java后端30天求职冲刺计划",
  "totalDays": 30,
  "weeklyGoals": [
    "第1周补齐分布式基础",
    "第2周强化项目表达与系统设计"
  ],
  "dailyTasks": [
    {
      "day": 1,
      "task": "学习分布式事务常见方案并整理对比表",
      "durationMinutes": 90,
      "taskType": "learn",
      "deliverable": "一页方案对比笔记"
    }
  ],
  "milestones": [
    {
      "day": 7,
      "goal": "完成P0差距项首轮补齐"
    }
  ],
  "riskAndFallback": [
    "若本周时长不足，优先完成P0任务，P1顺延"
  ]
}
```

### Coze提示词模板（可直接用）

```text
你是求职训练营教练，请为候选人制定可执行提升计划。

输入：
- gapsJson: {{gapsJson}}
- targetDays: {{targetDays}}
- weeklyHours: {{weeklyHours}}
- preferredLearningMode: {{preferredLearningMode}}
- nextInterviewDate: {{nextInterviewDate}}

要求：
1. 计划必须可执行，日任务时长合理（30~120分钟）。
2. 优先处理P0差距项。
3. 每项任务要有可交付物（deliverable）。
4. 输出字段：planName,totalDays,weeklyGoals,dailyTasks,milestones,riskAndFallback
5. 只输出严格JSON。
```

## 工作流4：面试复盘总结（JOB_BATTLE_INTERVIEW_REVIEW）

### 功能说明
对一次真实面试记录进行根因分析，给出高影响改进动作和 7 天修复计划。

### 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| interviewNotes | String | 是 | 面试后手工记录 |
| qaTranscriptJson | String | 否 | 问答记录（文本/转写结果） |
| interviewResult | String | 是 | pass/reject/pending |
| targetRole | String | 是 | 目标岗位 |
| nextInterviewDate | String | 否 | 下场面试日期 |

### 输出格式（严格 JSON）

```json
{
  "overallConclusion": "本次失败主要因系统设计深度不足与表达不聚焦",
  "rootCauses": [
    "回答偏概念，缺少项目证据",
    "高并发治理方案不成体系"
  ],
  "highImpactFixes": [
    {
      "issue": "系统设计题缺乏结构化",
      "action": "用固定模板演练10题（背景-目标-方案-权衡）",
      "deadline": "D+5",
      "metric": "每题5分钟内可讲清主方案和权衡"
    }
  ],
  "questionTypeWeakness": [
    {
      "type": "系统设计",
      "suggestion": "按容量估算→架构→瓶颈→优化顺序答题"
    }
  ],
  "next7DayPlan": [
    "Day1-2：整理2个项目的性能优化案例",
    "Day3-4：系统设计专题训练"
  ],
  "confidenceScore": 68
}
```

### Coze提示词模板（可直接用）

```text
你是资深面试教练，请对本次真实面试进行复盘。

输入：
- interviewNotes: {{interviewNotes}}
- qaTranscriptJson: {{qaTranscriptJson}}
- interviewResult: {{interviewResult}}
- targetRole: {{targetRole}}
- nextInterviewDate: {{nextInterviewDate}}

输出要求：
1. 给出失败/待提升的根因，不要泛泛而谈。
2. 给出高影响修复动作（issue/action/deadline/metric）。
3. 生成7天冲刺计划。
4. 输出confidenceScore（0-100）。
5. 只输出严格JSON。
```

## Coze搭建步骤（通用）

1. 登录 Coze，分别创建上述 4 个工作流。
2. 在「开始节点」配置对应输入参数（名称与本指南一致）。
3. 添加「大模型节点」，粘贴对应提示词模板。
4. 在「结束节点」输出 JSON（建议启用结构化约束/JSON Schema 校验）。
5. 用测试样例调试，确保输出稳定且字段完整。
6. 发布工作流并记录 workflowId。

## 后端接入建议

建议沿用当前项目模式：

1. 在 `CozeWorkflowEnum` 新增 4 个枚举。
2. 在 `xiaou-ai` 新增 `AiJobBattleService`（或并入现有服务）：
   - `parseJd(...)`
   - `matchResume(...)`
   - `generatePlan(...)`
   - `reviewInterview(...)`
3. 统一走 `CozeUtils.runWorkflow(...)`，并保留本地降级策略。
4. 统一解析 `{"output":"..."}` 包装响应，避免解析失败。

## 降级方案建议

当 Coze 不可用时，建议提供可用但简化的兜底：

- JD解析：关键词规则提取（正则+词典）。
- 简历匹配：基于技能交集计算基础分。
- 行动计划：按 P0/P1/P2 固定模板生成 7/14/30 天计划。
- 复盘总结：固定模板 + 关键字段回显。

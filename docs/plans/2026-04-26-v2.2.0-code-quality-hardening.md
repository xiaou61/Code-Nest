# v2.2.0 Code Quality Hardening

## 版本目标

`v2.2.0` 不新增业务功能，聚焦工程质量、可扩展性、规范化和可验证性。优化方向以低风险、高复用、高收益为优先，先打通统一异常、参数校验、前端 lint、版本基线和文档闭环，再逐步推进公共工具与大文件拆分。

## 已识别优化点

1. Controller 层重复 `try/catch` 需要收敛到全局异常处理。
2. `BusinessException` 默认构造存在空错误码/空消息风险。
3. 全局异常处理缺少请求体解析、请求头缺失、非法参数、上传大小等常见异常。
4. `@RequestBody` 请求多，参数校验覆盖不足。
5. CodePen 用户端和管理端 Controller 样板代码过多。
6. CodePen 请求 DTO 缺少基础边界约束。
7. 前端缺少 ESLint 配置入口。
8. 前端 lint 脚本使用已废弃的 `--ignore-path` 参数。
9. 前端历史代码存在 lint error，需要先清零 error 再治理 warning。
10. `README`、Maven、前端包版本需要统一到 `v2.2.0`。
11. 超大后端服务类需要拆分职责。
12. 超大 Vue 页面需要抽组件和 composable。
13. `ThreadPoolUtils`、`ConcurrentUtils` 职责过宽。
14. `RedisUtil` 中异常后返回 `null/false/0` 的语义需要治理。
15. 存储策略实现重复度高。
16. SQL Mapper 中存在 `SELECT *`。
17. 配置中存在默认密钥和明文开发密码，需要脱敏策略。
18. 前端 `console.*` 数量较多，需要统一日志策略。
19. 测试覆盖集中在少数模块，需要补核心模块单元测试。
20. 构建工具缺少统一格式和质量门禁。
21. Sa-Token 登录模型存在过时 API 使用。
22. 公共 JSON/并发工具存在 unchecked 泛型噪声。
23. Lombok Builder 默认值缺少显式 `@Builder.Default`。
24. 子模块依赖版本需要回归父 POM 统一管理。
25. `pom-xml-flattened` 需要随版本发布同步刷新，避免版本漂移。
26. 前端生产构建缺少 vendor 拆包策略，页面 chunk 容易吞入重型依赖。
27. Markdown 高亮使用 `highlight.js` 整包导入，包体成本偏高。
28. Sass legacy JS API 提示需要切换到 Vite 支持的现代 Sass API。
29. 后端 raw `Map.class/List.class` 解析和强转需要收敛为类型安全转换。
30. go-judge 客户端响应解析缺少泛型约束，异常数据可能在运行期才暴露。
31. 文件迁移成功后删除源文件应保留原始存储路径，避免更新实体后误用新路径。
32. 弱类型对象到 `Map` 的转换逻辑散落在业务模块中，需要公共工具统一。
33. `xiaou-team` 每日统计 Mapper 字段与建表脚本不一致，存在运行期错列风险。
34. 生产 Mapper 的 `SELECT *` 应逐步替换为显式列清单，降低表结构变更影响面。
35. join 查询中的 `alias.*` 同样会放大表结构变更影响，需要收敛为别名显式列清单。

## 本批已落地范围

- 增强 `GlobalExceptionHandler` 常见异常覆盖。
- 修正 `BusinessException` 默认构造和消息兜底。
- 重构 CodePen 用户端与管理端 Controller，减少重复异常包装。
- 为 CodePen DTO 增加基础参数校验。
- 新增 `.editorconfig`、双前端 `.eslintrc.cjs`。
- 修复前端 lint error/warning，使用户端和管理端均达到 `0 error / 0 warning`。
- 后端 Maven、前端 package、README 示例统一升级到 `v2.2.0`。
- 将 JWT secret 改为环境变量优先，保留开发兜底，并同步修正密钥生成工具输出说明。
- 清理 Sa-Token 过时登录 API，统一改为 `SaLoginParameter`。
- 优化 `RedisUtil` 失败返回、空 varargs、集合计数和 Redisson 过时 API 使用。
- 清理 `JsonUtils`、`ThreadPoolUtils` unchecked 泛型噪声。
- 为积分统计请求补齐 Builder 默认值，避免 Lombok 默认值失效。
- 去除 `xiaou-system` 中重复声明的 JJWT 版本，回归父 POM 统一依赖管理。
- 同步刷新根模块与子模块 `pom-xml-flattened`，统一发布版本号。
- 为双前端 Vite 构建增加命名 vendor 拆包和 1200KB chunk 预算。
- 将 Markdown 高亮从 `highlight.js` 整包改为 core + 常用语言按需注册。
- 将 Sass 预处理切换到 Vite `api: 'modern'`，移除 legacy JS API 构建噪声。
- 将文件存储、博客配置、聊天 WebSocket、迁移任务和 go-judge 客户端中的 raw Map/List 解析改为类型安全转换。
- 为 `JsonUtils` 增加字符串 Map 解析方法，减少业务模块重复手写 JSON 转换。
- 修正文件移动流程中的源文件删除路径，避免实体更新后用目标路径删除源存储文件。
- 将弱类型对象到 `Map<String, Object>` / `Map<String, String>` 的转换收敛到 `JsonUtils`，减少 WebSocket、迁移任务和 go-judge 客户端重复 helper。
- 将 `xiaou-sql-optimizer` 查询记录 Mapper 从 `SELECT *` 改为复用显式列清单。
- 将 `xiaou-team` 任务、讨论、打卡、每日统计 Mapper 的 `SELECT *` 改为显式列清单，并修正每日统计字段名与建表脚本不一致问题。
- 将 `xiaou-oj` 题目、标签、题解、测试用例、提交、赛事、参赛和赛事题目 Mapper 的通配列查询改为显式列清单。
- 将 `xiaou-team` 加入小组列表、`xiaou-community` 评论 join 查询、`xiaou-interview` 题单/题目/掌握度/收藏 join 查询中的 `alias.*` 改为显式别名列清单。

## 已知剩余风险

- Maven 仍提示 `rdc` profile 不存在，这是历史构建环境提示，本次未新增。
- 本批后端验证使用 `-DskipTests`，覆盖重点是编译、依赖和静态质量门禁；后续应补 CodePen 参数校验与异常出口测试。
- 双端前端构建已无 Sass legacy 与大 chunk 警告；后续如果继续瘦包，可推进 Element Plus 按需导入和 G6/ECharts 页面级懒加载。

## 后续建议

- 将 Element Plus 全量注册、图谱/图表依赖懒加载作为下一轮前端瘦包治理。
- 将 `ThreadPoolUtils`、`ConcurrentUtils` 按执行器、重试、批处理、熔断等职责拆分。
- 将 `RedisUtil` 的空值返回改为显式结果语义或抛出业务异常。
- 为 CodePen Controller 增加 WebMvcTest，锁定参数校验和错误码行为。
